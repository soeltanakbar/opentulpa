# OpenTulpa Chat Idea Bank

Use this as a quick list of things you can ask OpenTulpa to do through chat.

## Important Notes

- You can pass env vars through chat during setup.
- OpenTulpa stores generated scripts/config in the `tulpa_stuff` sandbox.
- External write actions still use approval guardrails.

## High-Value Ideas

- Beeper inbox copilot:
  connect your local Beeper connector, summarize unread/unanswered chats from the last 24h, and draft replies.

- Gmail triage:
  summarize unread/important emails, extract action items, and schedule follow-up reminders.

- API integration builder:
  provide API keys in chat, generate integration scripts, and compose reusable workflows.

- Daily executive brief:
  aggregate selected sources/channels and send a concise morning briefing.

- Sales/research assistant:
  collect leads/signals, enrich context, and draft outreach variants in your style.

- Incident helper:
  detect failures from logs/events, summarize impact, and suggest the next recovery actions.

- Content automation:
  gather trends, draft posts, and queue/schedule publishing flows with approval before external posting.

- Personal operations:
  convert repeated requests into reusable skills and routines with your preferred format/persona.
